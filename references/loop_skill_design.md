# Loop 併用スキル設計ガイドライン

`/loop <interval> /<skill>` で定期実行するスキルを作成する際の設計原則。

## 冒頭テンプレート

スキル本文の冒頭に以下を明記し、Claude が途中で質問や確認を挟まないようにする:

```
`/loop <interval> /<skill>` での定期実行を想定。
1回の実行で調査→報告→状態更新まで自律的に完了する。
ユーザーへの質問や確認は行わない。
破壊的操作（ファイル削除・移動・コード変更）は行わず、報告のみ。
```

## 設計原則

### 1. 完全自律実行（最重要）
- ユーザー応答を待つ分岐を入れない
- 「提案のみ・実施しない」か「安全な範囲で自動実施」のどちらかに決める

### 2. 冪等性
- 同じ状態で2回実行しても壊れないこと
- 状態ファイルの書き込みが途中で失敗しても次回復旧できる設計にする（例: index が範囲外なら 0 にリセット）

### 3. 出力の簡潔さ
- 定期実行で長文が出ると会話が埋まる
- 「問題なし」の場合は1行で済ませ、問題がある場合だけ詳細を出す

### 4. 破壊的操作の禁止
- ユーザー確認なしで実行されるので、ファイル削除・移動・コード変更はしない
- 「報告・提案のみ」が安全なデフォルト

### 5. 状態管理
- `.claude/` 配下に JSON で状態を持つ（例: `.claude/<skill>_state.json`）
- 状態ファイルが存在しない場合のデフォルト初期化を必ず書く
- index 系の値が範囲外になった場合のリセット処理を入れる

### 6. 実行時間
- loop 間隔より十分短く終わること
- 重い処理（全ファイル Read 等）は避け、Glob + 必要なファイルだけ Read にとどめる

### 7. パスの書き方
- 絶対パスをスキル本文にハードコードしない
- プロジェクトルートからの相対パスで記述する
