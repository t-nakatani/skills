# Pause Session

セッションを一時停止し、後から別セッションで復帰するためのスキル。
`/session-review` が「振り返り」なのに対し、こちらは**復帰**を目的としたスナップショット。

## モード判定

引数やユーザーの発言から自動判定する:

| ユーザーの意図 | モード |
|---------------|--------|
| 一時停止したい / pause | **Pause** |
| 復帰したい / resume / restore | **Restore** |
| 一覧を見たい / list | **List** |
| 引数なし（デフォルト） | **Pause** |

---

## Pause モード

現在のセッションのコンテキストを復帰用マークダウンとして保存する。

### 手順

1. **コンテキスト抽出**: 現在の会話を振り返り、以下を抽出する
   - タスク: 何をしていたか（ユーザーの最初のリクエスト〜現在の作業）
   - 進捗: 何が完了し、何が未完了か
   - 作業ファイル: 編集・参照した主要ファイル
   - 判断・決定事項: セッション中に下した設計判断や方針
   - 未解決事項: ブロッカー、未回答の質問、保留にした判断
   - 次のアクション: 復帰後にやるべきこと

2. **Git 状態の記録**: 現在のブランチ名と未コミットの変更概要を記録する

3. **トピック決定**: AskUserQuestion でトピック名（英語 kebab-case, 2-4語）を確認する
   - 提案も出す（例: `boros-market-maker-refactor`）

4. **ファイル保存**: `.claude/sessions/paused/YYYY-MM-DD_HH-MM_<topic>.md` に保存する

5. **確認**: 保存パスを表示し、セッションを閉じるかどうか確認する

### 保存フォーマット

```markdown
---
topic: <トピック名>
paused_at: "YYYY-MM-DD HH:MM"
branch: <git branch>
status: paused
---

## タスク

<何をしていたか。ユーザーのリクエストを含む具体的な記述>

## 進捗

- [x] 完了した作業
- [ ] 未完了の作業

## 作業ファイル

- `path/to/file.rs` — 変更内容の簡潔な説明
- `path/to/other.py` — 参照した理由

## 判断・決定事項

- <セッション中に決めたこと。なぜそう決めたかも含む>

## 未解決事項

- <ブロッカー、保留中の質問、未確定の方針>

## 次のアクション

1. <復帰後に最初にやること>
2. <その次にやること>

## コンテキスト補足

<新しいセッションの Claude が理解に必要な背景情報。
プロジェクト構造、関連するドメイン知識、特殊な制約など。
コードや git log から読み取れる情報は省略してよい。>
```

### ポイント

- **復帰する Claude が読む**前提で書く。現セッションの文脈に依存する曖昧な表現を避ける
- コードベースを読めば分かる情報（関数シグネチャなど）は省略し、**意図・判断・文脈**に集中する
- 「なぜこのアプローチを選んだか」「何を試して何がダメだったか」は特に重要
- 進捗はチェックリスト形式で、完了/未完了が一目で分かるようにする
- 重要な発見事項（バグ、脆弱性、設計上の問題など）は、外部ファイルに詳細があっても**スナップショット内に概要を1行書く**。復帰 Claude がファイルを開く前に状況を把握できるようにする

## Restore モード

一時停止中のセッションのコンテキストを読み込み、作業を継続する。

### 手順

1. `.claude/sessions/paused/` 内のファイル一覧を取得する
2. 各ファイルの frontmatter（topic, paused_at, status）を読み、一覧表示する
3. AskUserQuestion でどのセッションを復帰するか選択させる
4. 選択されたファイルを読み込む
5. Git ブランチが記載されている場合、現在のブランチと一致するか確認する（不一致なら警告）
6. コンテキストを要約して提示し、「次のアクション」から作業を再開する
7. frontmatter の `status` を `restored` に更新する

### 復帰時の振る舞い

- スナップショットの「次のアクション」をそのまま実行するのではなく、まず現在の状態を確認する
  - git status、ファイルの現在の内容を確認し、スナップショット以降に変更がないか確認
- 状況に変化があれば、ユーザーに確認してから作業を再開する
- 復帰は新しいセッションで行うため、以前のツール呼び出し結果やファイル内容は保持されていない。必要なファイルは改めて読む

## List モード

一時停止中のセッション一覧を表示する。

### 手順

1. `.claude/sessions/paused/` 内の `.md` ファイルを取得
2. 各ファイルの frontmatter を読み、テーブル形式で表示:

```
| # | 日時 | トピック | ブランチ | 状態 |
|---|------|---------|---------|------|
| 1 | 2026-03-15 14:30 | boros-mm-refactor | feature/mm | paused |
| 2 | 2026-03-14 22:00 | xyz-mm-obi-signal | main | restored |
```

3. `restored` 済みのセッションがある場合、削除するか確認する

---

## ルール

- 日本語で出力する
- frontmatter の日時は JST（Asia/Tokyo）
- トピック名は英語 kebab-case（2-4語）
- `status` は `paused` / `restored` の2値
- 復帰済みファイルの削除はユーザー確認の上で行う（自動削除しない）
